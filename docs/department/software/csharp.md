## C#是什么？

C#（读作 C Sharp 不是 C 井）是一种由微软（Microsoft）开发的现代化、面向对象的高级编程语言，最初在 2000 年发布，是 .NET 平台的主要开发语言之一。C# 结合了 C++ 的高性能和 Java 的简洁性，广泛应用于桌面应用、Web 开发、移动应用、游戏开发（尤其是使用 Unity）、后端服务等领域。

## 学习 C# 能做什么？

C#是一门现代、功能强大且结构清晰的编程语言，在多个领域都有广泛应用：

### 🖥️ Windows 桌面应用开发

- **WinForm**：经典的 Windows 窗体应用程序（小时候抱过我）。虽然界面相对简朴，但学习成本低，开发效率高，适合快速构建企业内部工具、数据管理系统等实用性应用。许多老牌软件至今仍在使用 WinForm 技术栈。

- **WPF（Windows Presentation Foundation）**：功能强大的现代化桌面应用程序框架，支持数据绑定、样式模板、动画效果等丰富特性。采用 XAML 声明式 UI 设计，能够创建美观的用户界面。适合开发复杂的桌面应用，如专业软件、设计工具等。

- **WinUI 3**：微软最新推出的现代化 Windows 桌面 UI 框架，融合了 UWP 和传统桌面应用的优势。支持最新的 Windows 11 设计语言（Fluent Design），提供原生性能和现代化体验，是 Windows 桌面应用开发的未来发展方向。

### 🌐 Web 开发

基于 **ASP.NET Core** 这一高性能、跨平台的 Web 框架构建现代应用：

- **Razor Pages**：轻量级的页面驱动开发模式，将页面逻辑和视图紧密结合。适合内容驱动的网站、企业官网、博客系统等场景，学习曲线平缓，开发效率高。

- **MVC （Model-View-Controller）**：经典的分层架构模式，通过模型、视图、控制器的分离实现关注点分离。适用于大型复杂网站、企业级应用、RESTful API 开发，代码结构清晰，便于团队协作和维护。

- **Blazor**：革命性的 Web 开发框架，让 C# 开发者能够使用熟悉的语言编写前端代码，告别 JavaScript。支持 Server-side 和 WebAssembly 两种运行模式，特别适合 .NET 技术栈团队快速构建交互式 Web 应用。

### 🎯 跨平台框架

- **.NET MAUI**：微软官方维护的跨平台框架，是 Xamarin 的继任者。一套代码可以构建 Windows、macOS、iOS、Android 应用，大大降低了跨平台开发成本。虽然目前生态仍在完善中（半死不活），但对于需要快速覆盖多平台的项目来说是不错的选择。

- **Avalonia**：社区驱动的开源跨平台 UI 框架，支持 Windows、macOS、Linux 等平台。特别值得称道的是其对 Linux 桌面环境的优秀支持，能够提供真正的原生体验。语法与 WPF 相似，学习成本较低。

### 🎮 游戏开发

- **Unity 引擎集成**：C# 作为 Unity 引擎的主要脚本语言，广泛应用于 2D/3D 游戏开发。从独立游戏到 AAA 大作，从移动游戏到 VR/AR 应用，Unity + C# 组合为游戏开发者提供了强大而灵活的开发环境。丰富的资源商店和活跃的社区也是其重要优势。

### ☁️ 云原生

- **Microsoft Azure 深度集成**：C# 与 Azure 云服务深度集成，提供了完整的云原生开发体验。从 Azure Functions 无服务器计算，到 Azure App Service Web 应用托管，再到 Azure SQL Database 数据存储，整个开发部署流程无缝衔接。对于企业级云端应用开发来说，C# + Azure 是极具竞争力的技术组合。

## 如何学习 C#？

想了解更多和 C# 组方向相关的详细学习内容，请继续阅读下文。

我们无法给出上述所有方向的学习指引，若想了解请与对应组同学交流讨论。但是在这里，我们希望能给你一些软件开发方向的学习建议。

### 如果未来想继续走计算机相关方向（考研除外），下面是一些忠告

- **学英语，用英语**，尤其是计算机相关的表达。不要滥用全文翻译，这对学英语没有帮助。可以使用划词翻译。推荐使用浏览器插件：[沉浸式翻译](https://immersivetranslate.com/)。微软式中文会显著激发你学习英语的热情，因为其中文文档往往需要进行中译英后才能理解。如果坚持使用中文（暂且不提有相当多的文档不会提供中文版本），你可能会见到以下翻译：

  > [此处原文档包含微软中文文档翻译的搞笑截图示例]

- **不要使用百度**，可以使用 Google、Duckduckgo 或者 Bing（国际版）。

- **尽可能减少使用 CSDN 等野生的中文互联网资料**，上面的许多内容过时、片面甚至错误。如果觉得 CSDN 好，可能只是见识太少了而且不会英语，没有见过更优质的内容。（参考实验报告可以用）

- **不提供恰当的资助就不要随便向陌生人问低级问题**，所有人都很忙而且没有回答的义务。可以先去问搜索引擎。（甚至问 GPT，但是需要额外注意信息正确性）

**总的来说，推荐大家努力去尝试适应全英文上网，可以先从只使用英文关键词来搜索开始。**

### 下面是在本组学习软件开发的建议

- **大部分情况下，微软为 C#提供的说明文档足以指导你学习相关概念**，请务必学会查找和阅读官方文档。

- **不要用应试思维学习开发**，刷算法题对提升开发能力没有任何帮助。最有用的学习方式是：以需求为原动力，在编写代码的实践中学习，通过查找资料、调整方案、解决问题的流程来提升开发水平，积累项目经验。

- **课堂内容严重脱节于实际生产**，如果想从事计算机方向，请务必拥有自行查找资料并自学的能力。网络上计算机方向的免费学习资料非常多，学会查找、筛选、学习、应用。

- **计算机大类专业（计算机、通信、自动化、物联网）并不会限制你的就业方向**，以自己的兴趣为导向，才能让自己有动力学习。

- **我们的授课内容并不是你要学习的全部知识**，更多地是提供学习指引，告诉你往什么方向学习。
