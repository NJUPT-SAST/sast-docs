## 前端组 😍

### 关于前端

传统前端开发是指创建网站或Web应用用户界面的过程，主要关注用户直接交互部分，其核心职责包括：

- 将UI/UX设计稿转化为可交互网页
- 优化用户体验以适配不同设备
- 确保跨浏览器兼容性
- 通过代码分割等策略提升性能

**核心技术栈**以HTML、CSS、JavaScript为基础：

- **HTML**：构建页面结构
- **CSS**：负责样式设计，可结合Sass/Less等扩展技术
- **JavaScript**：实现交互逻辑

主流框架如React、Vue.js、Angular及jQuery、Lodash等辅助库广泛应用。

**工程化工具**包括：

- 构建工具：Webpack、Vite等
- 包管理工具：npm、Yarn等

**开发流程**涵盖：需求分析、设计协作、编码实现、测试调试及部署上线等环节。

### 当前发展趋势

- 🌐 跨端开发（如React Native、Flutter）
- 🤖 AI与前端结合（如GitHub Copilot）
- 🔗 Web3与元宇宙开发（基于区块链的DApp、3D虚拟空间）
- 🛠️ 低代码/无代码平台应用

- 可以继续从事传统的web开发（如bilibili网页端的开发）

- 功利性的角度来看，前端目前的岗位需求量大

## 后端组 😎

### 关于后端

后端开发是指在服务器上运行的、不涉及用户界面的部分的开发工作。它主要负责：

- 处理应用程序的业务逻辑
- 数据存储和管理
- 与前端的交互

后端开发者使用各种编程语言和框架来实现这些功能，并确保应用程序的高效、安全和稳定运行。

> 💡 说白了后端开发干的就是用户看不见的界面。

### 后端的主要职责

1. **实现业务逻辑**：后端开发者负责实现应用程序的核心功能，如用户注册、登录、数据查询等。这些功能通常通过编写服务器端代码来实现。

2. **数据存储和管理**：后端开发者需要设计和管理数据库，以确保数据的持久性和一致性。

3. **API开发**：后端开发者需要创建API接口，供前端调用，以实现数据的传输和交互。

4. **优化和维护**：后端开发者需要不断优化代码和系统架构，以提高应用程序的性能和可维护性。

### 后端开发的技术栈

- **编程语言**：常用的后端编程语言有Java、Golang、PHP等。
- **数据库**：后端开发者需要熟悉SQL语言和数据库管理系统，如MySQL、PostgreSQL、MongoDB等。
- **框架和库**：后端开发者常用的框架有Spring（Java）、Django（Python）、Laravel（PHP）等。
- **服务器和部署**：后端开发者需要了解服务器的配置和管理，如Nginx、Apache等，以及容器化技术如Docker。

### 如何学习后端

后端工程师的核心职责在于**数据架构设计与业务逻辑实现**。持续学习能力是关键。

- 一方面要把数据怎么存、业务逻辑怎么写这些基础打牢
- 一方面又要紧跟前沿技术迭代，比如现在都在谈云原生、大模型等，这些都需要你不断的学习
